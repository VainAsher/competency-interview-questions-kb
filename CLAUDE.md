# CLAUDE.md — house rules for this knowledge base

**Subject:** Competency-based interview questions and best practice — UK & EU.
**Audience:** Both candidates preparing for interviews and interviewers/HR
designing and scoring them.
**Genre:** Reference — standards-backed, source-cited fact, with clearly
labelled coaching guidance.

## The one binding rule

**Never let a judgement wear the costume of a fact.**

- The **evidence layer** (`Framework Context`, `Core Reference`,
  `Assessor Perspective`, `Legal & Fairness Considerations`) cites every
  factual claim to an official framework document, government/EU-institution
  guidance, a regulator or statute, a professional body, or a named, dated
  authoritative source — with a working link.
- The **coaching layer** (`Model Answer Guidance`, `Common Pitfalls`,
  `Practice Exercises`) is labelled guidance. It points back to the evidence
  and never introduces a new "fact".
- Every **Question Bank** entry carries provenance: *(sourced …)* [n] for
  published questions, *(composed)* for questions written here in the style of
  a cited framework. A composed question is never attributed to an employer or
  presented as a real/leaked interview question.

## Process rules

- **One writer per file.** Workers write only their own document and never edit
  shared files (`MASTER_INDEX.md`, `GLOSSARY.md`, `SOURCE_REGISTRY.md`, …) or
  run git. The orchestrator owns all shared files, git, and releases.
- **Gates before freeze** — all green: `scripts/validate.py`,
  `scripts/audit_genre.py --strict`, `markdownlint-cli2`,
  `scripts/check_links.py` (0 dead links, independently re-verified by the
  orchestrator, not the worker's word).
- **The human gate is a hard stop.** No cluster is frozen/tagged without human
  approval, unless the user has granted a standing "auto-approve on green QA"
  mandate.
- **Freeze/version/release.** A fact change cuts a new version with a revision
  note — never a silent edit.
- **Sources:** follow the ranked priority order in `SOURCE_REGISTRY.md`. Never
  fabricate or guess a URL; if you cannot open it, do not cite it.

## Layout

- `docs/01-competencies/ … docs/06-best-practice/` — the documents, one folder
  per category.
- `templates/document_template.md` — the 20-section skeleton every document
  copies; `scripts/validate.py` enforces it.
- `prompts/worker_contract.md` — handed verbatim to every worker agent.
- `exports/` — generated artefacts (`build_graph.py`, `build_rag.py`); never
  hand-edit. `mkdocs.yml` and `docs/index.md` are generated by
  `build_site.py`; never hand-edit.
